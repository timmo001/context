`context` gives agents and humans one command for the repository facts they usually gather with many shell calls.

<CardGrid>
  <Card title="Git context" icon="code-branch">
    Branch, pull request, working-tree, diff, and recent-commit context in one
    deterministic snapshot.
  </Card>
  <Card title="Stack detection" icon="magnifier">
    Language, ecosystem, tooling, and framework detection from repository files,
    with no LLM and no external scanners.
  </Card>
  <Card title="MCP server" icon="server">
    The same producers exposed over stdio as read-only MCP tools and resources
    for agent harnesses.
  </Card>
  <Card title="Generated references" icon="open-book">
    CLI help, shell completions, and MCP tool docs generated from the same
    source metadata.
  </Card>
</CardGrid>
